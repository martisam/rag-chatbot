RAG Chatbot - Query Flow

End-to-end lifecycle of a user query - from browser to Claude API to ChromaDB and back

Browser / Frontend
Backend (Python)
External Services
1
User sends message
script.js - sendMessage()
Input captured, input disabled, loading bubble shown. POST /api/query with {query, session_id}
▼
2
FastAPI receives request
app.py - query_documents()
Validates body via Pydantic. Creates session if session_id is null. Calls rag_system.query()
▼
3
RAGSystem orchestrates
rag_system.py - query()
Wraps prompt. Fetches conversation history from SessionManager. Calls ai_generator.generate_response() with tool definitions.
▼
4
Claude API - Call #1
ai_generator.py - generate_response()
Sends system prompt + query + search_course_content tool. tool_choice: auto
▼
5
Claude decides
stop_reason check
Course-specific - tool_use → run search
General knowledge - end_turn → skip to synthesis
▼
6
Tool execution
ai_generator.py - _handle_tool_execution()
Appends Claude's tool_use block to message history. Calls tool_manager.execute_tool()
▼
7
CourseSearchTool
search_tools.py - execute()
Resolves course name via course_catalog (fuzzy match). Queries course_content with optional lesson filter.
▼
8
ChromaDB search
vector_store.py - search()
Cosine similarity search using all-MiniLM-L6-v2. Returns top 5 chunks with course/lesson metadata.
▼
9
Claude API - Call #2
ai_generator.py
Synthesizes retrieved chunks. No tools passed. Returns clean markdown answer.
▼
10
RAGSystem cleanup
rag_system.py
Calls get_last_sources(), resets source tracking. Calls session_manager.add_exchange() to persist history.
▼
11
FastAPI returns response
app.py - QueryResponse
Returns {answer, sources, session_id} as JSON.
▼
12
Frontend renders answer
script.js - addMessage()
Loading bubble removed. Answer rendered via marked.parse(). Sources shown in collapsible panel. Input re-enabled.
